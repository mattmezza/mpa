Browser automation (optional)

Let the agent read JS-heavy pages and act on sites via a headless browser (Chromium). Off by default — you can enable it later and add per-domain rules in Settings → Tools.

What works: most sites, including logged-in sessions you set up once. May be blocked: sites behind strong bot-management or interactive challenges.

First-time login is guided and mobile-followable: ask the agent to open a site, it sends a screenshot, you provide credentials and approve from your phone, and the session is saved for reuse. Reading runs without asking; actions ask for approval each time.
