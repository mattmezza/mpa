{# Basic identity fields #}

Identity

The agent's name — how it introduces itself.

{# Character #}

Character & tone

Defines who the agent is and how it behaves — identity, tone, decision-making style, language rules. Markdown format.

{# Default-identity account bindings (#110) #}

Email, calendar & contacts accounts

Which accounts the default agent (when no agent is active) may use, and at what level. None = no binding. Read = read only. Read & write = also send / create. Pick one email account as the sender identity. Leave everything on None to keep the legacy behaviour (the default agent may use any configured account). Add accounts in the Email / Calendar / Contacts tabs.

{# Voice #}

Voice

Configure speech-to-text (STT) and text-to-speech (TTS) used for voice messages.

Larger models are more accurate but slower.

Kokoro runs on-device (model bundled by default); it falls back to Edge TTS if the model isn't available. Restart after changing.

{# Edge voice #}

Use Edge TTS voice names. Example: en-US-AvaNeural.

{# Kokoro default voice #}

Voice prefix sets the default language (af/am = English, ff = French, jf/jm = Japanese, …). Preview needs the Kokoro backend running.

{# Preview — type text, pick a language (Kokoro), hear the selected voice #}

Language sets pronunciation independently of the voice — e.g. an Italian voice can read English text.
