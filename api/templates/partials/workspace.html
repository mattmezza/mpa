Workspace

A coding harness that lets the assistant read, write, edit, search, and run commands on a real codebase directly — instead of dictating every change in chat for you to apply by hand.

Coding harness

File tools (read_file, write_file, edit_file, list_dir, grep, run_command_in_dir) confined to the directory below. Reads run without asking; writes and commands ask for your approval each time. Paths that escape the directory (via .. or a symlink) are blocked. Disabled, or with a blank directory, the tools are hidden from the assistant entirely. Newly enabled, they appear from the next /new conversation in session history mode.

Absolute path (or ~) to the only directory the assistant may touch. Leave blank to keep the harness inert. There is no default — the assistant cannot reach the filesystem until you point it at one.
