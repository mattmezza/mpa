Workspace

A coding harness that lets the assistant read, write, edit, search, and run commands on a real codebase directly — instead of dictating every change in chat for you to apply by hand.

Coding harness

File tools (read_file, write_file, edit_file, list_dir, grep) confined to the directory below — paths escaping it (via .. or a symlink) are blocked. Reads run without asking; writes ask for your approval. run_command_in_dir runs a shell command in a directory below (linters, tests, builds): only its working directory is confined — the command itself runs with full process privileges, gated by approval (it shares the Permissions rules for run_command). Disabled, or with a blank directory, the tools are hidden from the assistant entirely. Changes take effect on the assistant's next turn — no restart or /new needed.

Absolute path (or ~) to the only directory the assistant may touch. Leave blank to keep the harness inert. There is no default — the assistant cannot reach the filesystem until you point it at one.
