Comparisons are approximate and reflect core capabilities at time of writing. humux is designed as a personal AI — not a home automation hub or workflow tool.

Self-contained

One Docker container. No Kubernetes. No external databases. No cloud dependencies.

Multi-agent

Multiple agent identities with per-agent bots, accounts, skills, and permissions. Concurrent, isolated, non-looping.

Privacy-first

Encrypted vault. Local SQLite storage. No telemetry. Per-agent permissions. Offline voice option.

Extensible

Add capabilities by writing markdown files. No plugin architecture. No code required. Teach your agent new CLI tools.

Honest trade-offs

humux is purpose-built as a personal AI agent. It does some things differently:

Not a home automation hub

If you need IoT control, camera feeds, or Z-Wave integration, Home Assistant is the right tool. humux focuses on digital productivity.

Not a workflow engine

For complex multi-step integrations between SaaS tools, n8n excels. humux is an AI agent with tool access, not a node-based pipeline builder.

Not a general-purpose framework

AutoGPT and Assistants API let you build custom agent behaviors. humux ships with a complete set of integrations — but you can extend it via markdown skills.

Requires Docker + API keys

humux needs Docker and at least one LLM API key. It's self-hosted but not zero-infrastructure. The trade-off for full data control.

The right tool for personal AI

If you want a self-hosted AI that handles messaging, email, calendar, and more — all in one container — humux is built for you.
